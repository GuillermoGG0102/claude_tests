How to Create Claude Skills

A comprehensive guide to building powerful AI capabilities

< 500 LINES

Keep your skill code concise, focused, and maintainable

⭐ Mandatory Fields (Required)
📝 name
Concise identifier for your skill
"code-review"
"image-generator"
"api-debugger"
📖 description
What the skill does (its expertise)
"Analyzes code for bugs,
performance, and best
practices"
💡 Optional Fields (Customize)
🔧 allowed-tools
Tools your skill can access
["bash", "read",
"grep", "github"]
🤖 model
Claude model for this skill
"claude-opus-4-7"
"claude-sonnet-4-6"
📁 Recommended Directory Structure
my-skill/
├ skill.md (Skill definition with metadata)
├ scripts/ (Executable code)
├ main.py, helper.py, utils.py
├ references/ (Documentation)
├ API_GUIDE.md, EXAMPLES.md
└ assets/ (Images, templates, data)
├ icon.png, template.json
✅ Best Practices
✓
Focus on ONE primary task
✓
Write clear, descriptive names
✓
Only specify tools if needed
✓
Organize code logically
✓
Document with examples
✓
Stay under 500 lines
✓
Make reusable & composable
📋 Complete Example: Code Review Skill
name: code-review
description: Analyzes code for bugs, performance
issues, and best practices
allowed-tools:
- bash
- read
- grep
model: claude-opus-4-7
