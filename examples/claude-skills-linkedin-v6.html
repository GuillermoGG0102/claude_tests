How to Create Claude Skills

Simple, focused approach for building effective AI capabilities

🎯 The Core Principle
Have a clear picture of what task your skill will do. Understand it so well you can explain it in less than 500 lines of text. Write code that matches that understanding.
📋 Mandatory Elements
name
Unique identifier. Example: pr-description
description
What it does and when to use it. Clear, specific purpose.
⚙️ Optional Elements
allowed-tools
Tools the skill can use. Only specify if needed.
Add Only If Required
model
Specific Claude model. Only if different from default.
Add Only If Required
skill.md Format
--- name: pr-description description: Writes pull request descriptions. Use when creating a PR, writing a PR, or when user asks to summarize changes. --- # PR Description Skill When writing a PR description: 1. Run `git diff main...HEAD` 2. Write description: ## What [One sentence] ## Why [Brief context] ## Changes - Bullet points - Group related changes
📁 Directory Structure
skill-name/ ├── skill.md ← Metadata & instructions ├── scripts/ │ └── implementation.py ← Your code ├── references/ │ └── README.md ← Documentation └── assets/ └── sample_data.txt ← Test data
💡 Understand Your Task
Before writing code, be crystal clear about what the skill does and how it should behave.
📏 Under 500 Lines
Keep code focused and maintainable. One clear purpose. Not multi-purpose.
✍️ Transmit Your Vision
Describe your skill so clearly others (and future you) understand it immediately.
🎯 Single Purpose
One skill = one clear task. If it does multiple things, split into separate skills.
